## Region/version handling

**Regio-detectie**
- Bestand: `src/core/infrastructure/Address.cpp`
- Functie/class: `Address::LoadRegion()`
- Methode: leest `Process::GetTitleID()` en mapt naar `Address::Region` enum + `regionName`.
  Wordt 1× geladen bij plugin-start; daarna staat de regio in de statics `Address::regionId`/`regionName`.

**Ondersteunde regio's** (TitleID-defines in `Address.hpp`):
- [x] USA — `TID_USA 0x0004000000086300`
- [x] EUR — `TID_EUR 0x0004000000086400`
- [x] JPN — `TID_JPN 0x0004000000086200`
- [x] KOR — `TID_KOR 0x0004000000086500`
- [x] Welcome amiibo — aparte TitleIDs per regio (`TID_USAWA/EURWA/JPNWA/KORWA`), gemapt op
      `USAWA/EURWA/JPNWA/KORWA`. → **8 regio-kolommen totaal.**
- [x] anders: romhack **EUR Welcome Luxury** (`TID_EURWL 0x00040000004C5700`) → valt onder `EURWA`.

**Offsetstructuur**
- Bestand: `include/core/infrastructure/Addresses.hpp`
- Vorm: `static const u32 ADDRESSES[][8]` — elke rij is één logisch adres met 8 regio-varianten,
  kolomvolgorde = `Region`-enum: `{ USA, USAWA, EUR, EURWA, JPN, JPNWA, KOR, KORWA }`.

```cpp
static const u32 ADDRESSES[][8] = {
    { 0x1938BC, 0x193304, 0x1938DC, 0x1938DC, 0x19382C, 0x19382C, 0x19382C, 0x19382C }, //FUNCTION_FOR_SUBMIT
    // ...
    { 0x3055C8, 0x305630, 0x3055D4, 0x3055D4, 0x305688, 0x305688, 0x305640, 0x305640 }, //GETMONEYFUNC
};
```

**Hoe het opzoeken werkt** (`Address::Address(u32 address)`):
1. Als `regionId == USA` → gebruik het opgegeven (USA-)adres direct.
2. Anders: zoek de rij waar `row[0] == address` (USA-adres als sleutel) en pak `row[regionId]`.
3. Niet gevonden → behandel als dynamisch adres en gebruik de waarde ongewijzigd.

Methoden op `Address`: `.Write<T>()`, `.Patch()/.Unpatch()`, `.Call<T>(args...)`, `.MoveOffset(off)`.

## Aanbevolen manier om nieuwe offsets toe te voegen
- Voeg een **nieuwe rij** toe aan `ADDRESSES[][8]` met het **USA-adres in kolom 0** als sleutel,
  plus de 7 andere regio-adressen. Gebruik in code altijd `Address(USA_ADDR)` — de regio-laag
  kiest automatisch het juiste adres. **Nooit** een los hardcoded adres zonder rij.
- Save-data hoeven we vaak niet per regio te offsetten: via `Player::GetSaveData()`/`Town::GetSaveData()`
  werk je met C++-structs (`ACNL_Player`/`ACNL_TownData`) waarvan de veld-offsets regio-onafhankelijk zijn.
  Alleen *functie-adressen* (zoals encrypt/decrypt, save-offset getters) lopen via `ADDRESSES`.
