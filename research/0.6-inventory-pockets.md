# Stap 0.6 — Inventory/pockets-structuur (resultaat)

> Bron: code-research (`Inventory.hpp`/`.cpp`, `Item.hpp`, `GameStructs.hpp`)

## Inventory/pockets

**Aantal slots**
- **16 pocket-slots** (`Item Inventory[16]` @0x6C70 in `ACNL_Player`). Bevestigd door macro
  `IsItemSlot(X) = (X >= 0x00 && X <= 0x0F)` en de loops `if (15 < slot) return`.
- Naast pockets bestaan: `IslandBox[40]`, `IslandInventory[16]`, `Dressers[180]` (closet, 60 lang elk),
  en de aparte `ACNL_Player_StorageBox` (secret storage). Voor turnips relevant: **pockets**.

**Slotstructuur** (`Item.hpp`)
```cpp
struct Item {        // 4 bytes, #pragma pack(1)
    u16 ID;          // item-ID
    u16 Flags;       // o.a. stack-aantal / quantity (te bevestigen in 0.7)
    // operator u32() == (Flags << 16) | ID
};
// In ACNL_Player:
Item Inventory[16];          //0x6C70
u8   InventoryItemLocks[16]; //0x6CB0  (per-slot lock)
```

**Slot lezen**
- Functie: `Inventory::ReadSlot(int slot, Item& item)` → `item = player->Inventory[slot]`.
- Bestand: `src/core/game_api/Inventory.cpp`
- Zoeken: `Inventory::GetNextItem(Item, u8& slot, bool ignoreFlag=false)` doorzoekt slot 0..15.
  Met `ignoreFlag=true` matcht het alleen op `ID` (negeert Flags/quantity) — handig om **elke turnip-stack**
  te vinden ongeacht aantal.

**Slot schrijven/leegmaken**
- Functie: `Inventory::WriteSlot(int slot, Item item, u8 lock=0)`:
  ```cpp
  player->Inventory[slot] = item;
  player->InventoryItemLocks[slot] = lock;
  ReloadIcons();
  ```
- Leegmaken = `WriteSlot(slot, {0x7FFE, 0})`. **Lege-slot-waarde `0x7FFE` is bevestigd uit de code**
  (`const u16 emptyID = 0x7FFE` in `Checks.cpp`; overal gebruikt om slots leeg te maken). Geen hardware nodig.
- Let op: `WriteSlot` weigert ongeldige items (`item.isValid(false)`), en roept `ReloadIcons()` aan
  zodat de UI bijwerkt.

## Item-ID type
- `u16 ID` (+ `u16 Flags`). Turnip-categorieën staan in `Item_Category` (`GameStructs.hpp`):
  - `Turnips` → ID-range **0x2283–0x228C**
  - `SpoiledTurnips` → **0x228D**

## Flags/quantity
- Stack-aantal zit naar verwachting in `Item.Flags` (zoals bij andere stackbare items). De `turnips()`
  cheat in `MoneyCodes.cpp` zet alleen **prijzen**, niet inventory-stacks, dus geeft hier geen uitsluitsel.
  → **Hardware-verificatie nodig (Stap 0.7)**: 10 vs 100 turnips leggen en `Flags` loggen.

## Bestaande helperfuncties (herbruikbaar)
- `Inventory::ReadSlot` / `WriteSlot` / `GetNextItem` / `GetSelectedSlot` / `GetHoveredSlot` / `ReloadIcons`
- `Item::GetName()`, `Item::isValid()`, `Item::searchByKeyword()`
- `Player::GetSaveData()` voor de player-base.

## Risico's
- `WriteSlot` schrijft direct in de save-RAM-struct; verkeerde slot/ongeldig item kan inventory
  corrumperen → altijd `isValid()` + slot-range check (0..15) + backup vóór test.
- Lock-byte niet vergeten te resetten bij leegmaken.
- Lege-slot-sentinel `0x7FFE` is bevestigd uit de code.

## Acceptatiecriterium — afgevinkt (m.u.v. hardware-detail)

      Flags-betekenis + lege-slot-waarde bevestigt.
