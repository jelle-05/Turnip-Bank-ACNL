# Stap 0.10 — Anti-overflow en payoutregels (resultaat)

> Bron: code-research + ACNL-limieten afgeleid uit slider-caps in `MoneyCodes.cpp`.

## Anti-overflow

**Max bank**
- **999.999.999 bells** (bank-slider cap in `bank()`; ACNL bankmaximum). Logisch 32-bit.

**Max wallet**
- **99.999 bells** (wallet-slider cap in `wallet()`). Niet primair relevant: payout gaat naar de bank,
  niet naar de wallet.

**Max turnip-prijs (per stuk)**
- Realistisch in-game tot ~660. De `turnips()`-cheat gebruikt een prijs-slider tot 99.999, dus als
  veiligheidsbovengrens voor berekening rekenen we met max prijs `99.999`.

**Max turnips in plugin-storage**
- Door ons te bepalen ontwerpkeuze (eigen SD-storage, Stap 0.11). Voorstel: harde cap zodat
  `turnips × prijs` nooit `BankMax` overschrijdt en geen overflow geeft.

**Berekeningstype**
- [x] **u64 nodig** voor de tussenberekening `payout = turnips * price`, ook al passen eindwaarden in u32.
  Reden: bij grote opslag (bv. 99.999 turnips × 99.999) = ~1.0e10 → buiten u32. Reken in u64, valideer,
  cast pas daarna terug.

**Gekozen gedrag bij overflow (v1)**
- [x] **Verkoop blokkeren** als de payout niet volledig in de bank past.
  - Past payout → volledig bedrag bij bank optellen + verkochte turnips uit plugin-storage halen.
  - Past niet → niets wijzigen, melding tonen: *"Bank limit would be exceeded"*.
- [ ] Gedeeltelijke verkoop tot banklimiet → optioneel later.

**Conceptcode**
```cpp
constexpr u64 BANK_MAX = 999999999ULL;

bool CanAddToBank(u64 currentBank, u64 payout, u64 maxBank = BANK_MAX) {
    return currentBank <= maxBank && payout <= (maxBank - currentBank);
}

// Verkoopflow (pseudo):
u64 cur     = (u64)Game::DecryptValue(&player->BankAmount);
u64 payout  = (u64)storedTurnips * (u64)currentPrice;   // u64 tussenstap
if (!CanAddToBank(cur, payout)) {
    MessageBox("Bank limit would be exceeded")();
    return; // niets aftrekken
}
Game::EncryptValue(&player->BankAmount, (int)(cur + payout));
storedTurnips = 0; // pas NA succesvolle bankupdate
```

## Belangrijke ontwerpregels
1. Bereken in **u64**, schrijf pas terug na de `CanAddToBank`-check.
2. Trek turnips **alleen** af na een geslaagde bankupdate (geen "geld weg, turnips ook weg"-race).
3. Bank lezen/schrijven uitsluitend via `Decrypt/EncryptValue` (secure value).
4. Cap stored turnips in eigen storage zodat een enkele payout `BANK_MAX` nooit kan overschrijden
   (anders is verkoop altijd geblokkeerd bij volle voorraad — UX-overweging).
