## Plugin storage

**Beschikbare file I/O helpers** (CTRPluginFramework)
- `File` + `Directory` klassen. Bewezen gebruik in dit project:
  - `File::Open(File& f, path, File::READ)` / `File::CREATE | File::WRITE` / `File::RWC | File::TRUNCATE`
  - `f.Read(buf, size)`, `f.Write(data, size)`, `f.GetSize()`
  - `File::Exists(path)`, `File::Remove(path)`
  - `Directory::IsExists(path)`, `Directory::Create(path)`, `Directory::Remove(path)`
  - `Directory::ListFiles(vec, ".ext")`, `Directory::ListDirectories(vec)`, `dir.OpenFile(...)`
- Returncodes via `File::SUCCESS` (vergelijken, niet aannemen).
- Padconventie: prefix **`E:/`** = root van de SD-kaart in CTRPF (zie `Files.h`, bv. `E:/Vapecord/...`).

**Bewezen schrijfpatroon** (uit `Config.cpp` / `Plugin_Color.cpp`)
```cpp
// schrijven (binair voorbeeld, Plugin_Color.cpp)
File file;
if (File::Open(file, path, File::CREATE | File::WRITE) == File::SUCCESS) {
    file.Write(data, dataSize);
}
// lezen
File file;
if (File::Open(file, path, File::READ) == File::SUCCESS) {
    file.Read(buf, size);
}
// directory aanmaken indien nodig
if (!Directory::IsExists(dir)) Directory::Create(dir);
```
- `Config.cpp` doet text read-modify-write van `config.ini`; `Plugin_Color.cpp` doet **binaire**
  struct-I/O — dat laatste is precies ons gebruiksmodel.

**Voorgesteld pad**
- Eigen, los van Vapecord's mappen, regio/speler-gescheiden:
  - `E:/3ds/ACNLTurnipBank/turnip_bank.dat` (één bestand), of
  - `E:/3ds/ACNLTurnipBank/<regio>/<townID>_<playerID>.dat` voor scheiding per speler/town.
  - Eerst `Directory::Create` van de bovenliggende mappen (recursief: per niveau checken/aanmaken,
    zoals `Config.cpp` doet voor `V_STANDARD` → `V_DATA` → regiodir).

**Bestandsformaat (concept v1)**
```cpp
#pragma pack(push,1)
struct TurnipBankFile {
    char magic[4];      // "ATB1"
    u32  version;       // 1
    u64  titleId;       // Process::GetTitleID() — regio/versie
    u16  townId;        // ACNL_TownData TID
    u16  playerId;      // PlayerID PID
    u64  storedTurnips; // aantal opgeslagen turnips
    u64  lastUpdate;    // timestamp (game-time of system-time)
    u32  checksum;      // CRC32 over alle voorgaande bytes
};
#pragma pack(pop)
```

**Atomic write mogelijk**
- [x] ja (handmatig): schrijf naar `turnip_bank.dat.tmp`, sluit, dan `File::Remove(dat)` +
  rename/replace. CTRPF biedt geen kant-en-klare rename overal; alternatief is
  "write tmp → verify read-back → overschrijf origineel". Veilig genoeg voor klein bestand.

## Backupstrategie
- Houd een tweede kopie `turnip_bank.bak`. Bij lezen: valideer `magic` + `checksum`; bij corruptie
  val terug op `.bak`. Schrijf-volgorde: nieuw `.dat` schrijven → bij succes `.bak` updaten.
- Voor de game-save zelf gebruiken we los `SaveBackupManager` (Stap 0.12), staat hier los van.
