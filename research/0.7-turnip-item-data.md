## Turnip item data

**Belangrijkste vondst: het aantal turnips zit in het ITEM-ID, niet in `Flags`.**
Elk ID 0x2283–0x228C is een vaste "bundle size". Eén slot met `ID=0x228C` = 100 turnips.

| Item-ID | Aantal |
|---|---|
| 0x2283 | 10 turnips |
| 0x2284 | 20 |
| 0x2285 | 30 |
| 0x2286 | 40 |
| 0x2287 | 50 |
| 0x2288 | 60 |
| 0x2289 | 70 |
| 0x228A | 80 |
| 0x228B | 90 |
| 0x228C | 100 |
| 0x228D | **Spoiled (rotte) turnip** — losse ID, geen bundle |

**Formule:** `aantal = (ID − 0x2283 + 1) × 10` voor 0x2283..0x228C (10..100, stappen van 10).

Normale turnip:
- Item-ID: 0x2283..0x228C (10..100)
- `Flags`: speelt **geen** rol voor de hoeveelheid (hoeveelheid zit volledig in het ID).

Rotte turnip:
- Item-ID: 0x228D (apart, geen aantal-codering)

Stackgedrag:
- [x] **aparte itemvariant** (ID = bundle size) — NIET quantity/flags.

**Praktische consequentie voor de plugin:** per slot zijn alleen veelvouden van 10 t/m 100 geldig.
Een willekeurig getal (bv. 37) kan niet in één slot. Voor "X turnips opslaan" moet je dus rekenen in
bundels, of meerdere slots / een eigen storage-teller gebruiken (sluit aan op 0.11: we tellen totaal
in ons eigen `.dat`, los van de game-slots).

## Save-parse (praktijktest-vervanging)
- De beschikbare `garden_plus.dat` had **geen turnips in de pockets** (inventory: tools + bells, rest `0x7FFE`),
  dus de Flags-codering kon niet empirisch uit déze save bevestigd worden.
- Wel bevestigd uit code: lege slot = `0x7FFE` (`const u16 emptyID = 0x7FFE`).
- Inventory-parse gevalideerd: 16 slots @0x6C70, plaintext `Item{ID,Flags}` — layout klopt.

## Conceptfuncties (nu bouwbaar)
```cpp
inline bool IsTurnip(Item it)        { return it.ID >= 0x2283 && it.ID <= 0x228C; }
inline bool IsSpoiledTurnip(Item it) { return it.ID == 0x228D; }
inline u32  GetTurnipAmount(Item it) { return IsTurnip(it) ? (it.ID - 0x2283 + 1) * 10 : 0; }
inline Item EmptySlot()              { return Item{0x7FFE, 0}; }
```
