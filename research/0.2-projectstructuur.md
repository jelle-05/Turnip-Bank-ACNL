## Projectstructuur

Het project is een CTRPluginFramework (CTRPF) plugin met een nette laag-scheiding:
`include/` (headers) spiegelt `src/` (implementatie). De codebase is feature-rijk en
bevat **al** functies voor vrijwel alles wat de Turnip Bank nodig heeft.

Belangrijke mappen:

| Map/bestand | Doel |
|---|---|
| `src/` + `include/` | Spiegelende headers/implementatie |
| `core/game_api/` | Game-abstracties: `Player`, `Town`, `Inventory`, `Item`, `Game`, `Save`, `House`, `NPC` |
| `core/game_api/GameStructs.hpp` | **Alle save-structs** (`ACNL_Player`, `ACNL_TownData`, `Item`, item-categorieën) — 2747 regels |
| `core/infrastructure/` | `Address`/`Addresses` (region-offsets), `SaveBackupManager`, `Language`, `CustomKeyboard`, `PluginUtils` |
| `platform/ctrpf/` | Menu-opbouw (`MenuCreate.cpp`), `MenuEntryIds`, folder-types, kleuren |
| `src/features/` | Concrete cheats, o.a. `MoneyCodes.cpp` (wallet/bank/coupon/**turnips**), `InventoryCodes/`, `SeedingCodes/`, `PlayerCodes/` |
| `include/features/cheats.hpp` | Centrale declaratie van alle feature-functies |
| `tools/localization/languages/*.json` | Vertalingen (nl/en/es) incl. bestaande turnip/bank-strings |
| `Makefile`, `3gx.ld`, `build.bat`/`build.sh` | Build → `.3gx` via `3gxtool`; vereist devkitARM + custom `libctrpfforvapecord` (dev-branch) |

### Feature-registratie/menu
- **Bestand:** `src/platform/ctrpf/MenuCreate.cpp`
- **Patroon:** een `MenuFolder` per categorie; entries via helper `CreateEntry(TextID, func, NOTE, MenuEntryId)` →
  `FOLDER->Append(CreateEntry(...))` → `menu->Append(FOLDER)`.
- Hotkeys per entry via `EntryWithHotkey(...)` / `AppendHotkeyNameKey`.
- Feature-functies gedeclareerd in `include/features/cheats.hpp`, geïmplementeerd in `src/features/...`.
- **Hier voegen we later de Turnip-Bank menu-items toe** (bv. een eigen `MenuFolder`), of we bouwen
  een eigen gestripte plugin die alleen deze entries registreert.

### Memory helpers (lezen/schrijven van waarden)
- **Bestand:** `src/core/game_api/Game.cpp`
  - `Game::EncryptValue(u64* pos, int value)` → roept game-functie `0x3036A4` aan (versleutelt geldwaarden).
  - `Game::DecryptValue(u64* pos)` → game-functie `0x3037DC` (leest versleutelde waarde uit).
  - **Belangrijk:** geld/turnip-prijzen staan als **`EncVal` (u64, versleuteld/"secure value")** in de save.
    Direct schrijven corrumpeert; je moet altijd via `EncryptValue`/`DecryptValue`. Dit beantwoordt
    de checksum-vraag van Stap 0.9 alvast: **ja, secure values, en de helper bestaat al.**
- Generieke RAM-toegang: CTRPF `Process::Read*/Write*`, en de `Address`-wrapper (zie hieronder).

### Offset/region handling
- **Bestanden:** `include/core/infrastructure/Address.hpp` + `Addresses.hpp`, impl. `src/core/infrastructure/Address.cpp`
- **Region-detectie:** `Address::LoadRegion()` leest `Process::GetTitleID()` en mapt naar 8 regio-IDs.
- **Offsettabel:** `static const u32 ADDRESSES[][8]` in `Addresses.hpp` — elke rij = 1 logisch adres,
  8 kolommen = 8 regio/versie-varianten. Kolom-index = `Address::regionId`.
- **Gebruik:** `Address a(0xUSA_ADDR);` → constructor zoekt de rij op USA-adres en pakt automatisch
  het juiste regio-adres. Methoden: `.Write<T>()`, `.Patch()`, `.Call<T>(args...)`, `.MoveOffset()`.
- Ondersteunde regio's (TitleIDs in `Address.hpp`):
  USA, USA-WelcomeAmiibo, EUR, EUR-WA (+ romhack EUR Welcome Luxury), JPN, JPN-WA, KOR, KOR-WA.
- **Strategie voor nieuwe offsets:** USA-adres als sleutel + de 7 andere regio-adressen als nieuwe rij
  in `ADDRESSES` toevoegen; nooit een los hardcoded adres gebruiken.

### Inventory/item handling
- **Bestanden:** `include/core/game_api/Inventory.hpp`, `src/core/game_api/Inventory.cpp`, `Item.hpp`
- `Inventory::ReadSlot(int slot, Item&)` / `WriteSlot(int slot, Item, u8 lock)` — lezen/schrijven via
  `Player::GetSaveData()->Inventory[slot]` (+ `InventoryItemLocks[slot]`), daarna `ReloadIcons()`.
- `Inventory::GetNextItem(Item, u8& slot, bool ignoreFlag)` — zoekt een item-slot.
- 16 pocket-slots: macro `IsItemSlot(X) = (X >= 0x00 && X <= 0x0F)`.
- **`Item`-struct (`Item.hpp`):** `{ u16 ID; u16 Flags; }`. Stack-aantal zit in `Flags` (te bevestigen
  in Stap 0.7). Item-categorieën incl. `Turnips` en `SpoiledTurnips` staan in `Item_Category`-enum.

### Money/bank handling
- **Bestand:** `src/features/MoneyCodes.cpp` — bevat al kant-en-klare voorbeelden:
  - `wallet()` → `Game::EncryptValue(&player->PocketMoney, value)`
  - `bank()`   → `Game::EncryptValue(&player->BankAmount, value)`  (slider tot 999.999.999)
  - `coupon()` → `MeowCoupons`, `medals()` → `MedalAmount`
  - `turnips()`→ schrijft `town->TurnipPrices[0..5]` (AM) en `[6..11]` (PM)
- **Structvelden (`GameStructs.hpp`):**
  - `ACNL_Player` (size `0xA480`): `EncVal BankAmount` (@0x6C2C), `EncVal PocketMoney` (@0x6FA8),
    `EncVal MedalAmount`, `EncVal MeowCoupons`, `Item Inventory[...]`, `InventoryItemLocks[...]`, `DebtAmount`.
  - `ACNL_TownData` (vanaf `0x2064`): `EncVal TurnipPrices[12]` (@0x6ADE0; 0-5 = AM, 6-11 = PM).
- **Speler-selectie:** `Player::GetSaveData(u8 idx=4)` → default actieve speler via `GetPlayerStatus(4)`;
  onderliggend `GetSpecificSave()`/`GetSaveOffset()` (game-functies `0x2FBA60`/`0x2FEB2C`).
  `Town::GetSaveData()` → town-data van de actieve speler.

### Save-backup (relevant voor Stap 0.12)
- `include/core/infrastructure/SaveBackupManager.hpp` — `BackupAllToCheckpoint(prefix, &name)`
  kopieert alle savebestanden naar de Checkpoint-map op SD met timestamp. Direct herbruikbaar als
  veiligheidsnet vóór risicovolle tests.
