
### Twee aanpassingen die nodig waren (nieuwere toolchain)
1. **`3gxtool` ontbrak** — geen standaard-devkitPro-tool. Geïnstalleerd via de ThePixellizerOSS
   pacman-database:
   ```sh
   # in C:\devkitPro\msys2\msys2_shell.bat
   # (database toevoegen aan /etc/pacman.conf — zie LibCTRPF README)
   pacman -Sy
   pacman -S --noconfirm 3gxtool      # 3gxtool 1.3-1 -> /opt/devkitpro/tools/bin/3gxtool
   ```
2. **`-Werror` in de LibCTRPF-build** brak op de nieuwe gcc (strengere warnings, o.a.
   `unused-but-set-variable` in `TextBox.cpp`). Opgelost door in
   `D:\jelle\libctrpfforvapecord\Library\Makefile` regel 80 `-Werror` → `-Wno-error` te zetten.
   Raakt alleen de **dependency**-build, niet de Vapecord-broncode. Warnings blijven zichtbaar.

> NB: De Vapecord-plugincode zelf is **ongewijzigd** gebouwd. Alleen de build-flags van de
> externe libctrpf-dependency zijn aangepast om met de actuele devkitARM te compileren.

## Aandachtspunten
- De build moet in de **devkitPro MSYS2-shell** draaien (`C:\devkitPro\msys2\msys2_shell.bat`), zodat
  `DEVKITPRO`/`DEVKITARM` gezet zijn en `make`/`arm-none-eabi-gcc`/`tar`/`bzip2` beschikbaar zijn.
  De gewone Git Bash heeft die omgeving niet automatisch.
- De Vapecord-Makefile verwacht de lib op `$(DEVKITPRO)/libctrpfforvapecord`; de lib-`make install`
  plaatst 'm daar exact, dus geen padaanpassing nodig.
- De lib bevat `libcwav`/`libncsnd` submodules in-repo (geen extra clone nodig).
