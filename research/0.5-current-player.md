## Current player / player base

**Actieve speler detectie**
- Bestand: `src/core/game_api/Player.cpp`
- Functie: `Player::GetSaveData(u8 PlayerIndex = 4)`
  ```cpp
  ACNL_Player *Player::GetSaveData(u8 PlayerIndex) {
      u32 *addr = (u32 *)GetSpecificSave(
          PlayerIndex >= 4 ? (u8)Player::GetPlayerStatus(4) : PlayerIndex);
      return (ACNL_Player *)addr;
  }
  ```
  - `PlayerIndex >= 4` (default) → "huidige/actieve" speler: `GetPlayerStatus(4)` levert de
    `PlayerStatus` van de geladen speler, die als index dient.
  - `PlayerIndex 0..3` → expliciet speler 1..4.

**Player index/type**
- Type: `enum class PlayerStatus : u8` (in `GameStructs.hpp`)
- Mogelijke waarden: `Town_00..Town_03 (0..3)`, `Isl_00..Isl_03 (4..7)`, `Empty (8)`.
- Praktisch: 4 spelerslots (0..3); waarde `Empty` = geen speler.

**Player base (hoe bepaald)**
- `Player::GetSpecificSave(u8 pPlayer)` → game-functie `0x2FBA60`, geeft het RAM-adres (base) van de
  save-struct van een specifieke speler.
- `Player::GetSaveOffset(u8 pIndex)` → game-functie `0x2FEB2C` (offset voor geladen spelers).
- `Player::GetPlayerStatus(u8 pPlayer)` → game-functie `0x2FF6CC`.
- Deze functie-adressen lopen via de `Address`-laag → **regio-onafhankelijk** te gebruiken.
- Regio-afhankelijk: [x] nee (data via structs) — alleen de getter-functieadressen zitten in `ADDRESSES`.

**Geldigheidscheck**
- `Player::SaveExists(ACNL_Player*)` → true als `PlayerData.PID != 0 && TownData.TID != 0`.
- Overal in de code wordt eerst `if(!player) return;` gedaan → zelfde patroon overnemen.

**Bank/inventory per speler of globaal?**
- **Inventory + Bank zijn per speler** (`ACNL_Player.Inventory[]`, `ACNL_Player.BankAmount`).
- **TurnipPrices zijn per town** (`ACNL_TownData.TurnipPrices[12]`), niet per speler.
  `Town::GetSaveData()` haalt de town van de actieve speler op.

**Kan wisselen tussen speler 1..4?**
- Ja: `Player::GetSaveData(0..3)` of `Player::Load(int pIndex)`.
